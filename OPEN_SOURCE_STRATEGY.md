## üîê Security Best Practices

### **1. Never Commit Secrets**

**Use `.gitignore`**:
```
*.key
*.pem
*.env
.env.*
*.secret
keystore/
wallets/
config.local.*
```

**Check Before Committing**:
```bash
# Scan for secrets
git secrets --scan
# Or use: trufflehog, gitleaks
```

### **2. Use Environment Variables**

**Instead of**:
```rust
let api_key = "secret123"; // ‚ùå BAD
```

**Do**:
```rust
let api_key = std::env::var("API_KEY")?; // ‚úÖ GOOD
```

### **3. Separate Public/Private Repos**

**Public Repo** (GitHub):
- Core code
- Documentation
- Client software

**Private Repo** (GitHub Private or Self-Hosted):
- Internal planning
- Security advisories
- Presale details (until launch)
- Business strategy

## üíº Legal Protections

### **1. Register Trademarks**

**What to Register**:
- "Mondoshawan" (name)
- "MSHW" (ticker)
- Logo (if distinctive)

**Cost**: $1,000-5,000 per trademark
**Time**: 6-12 months
**Protection**: 10 years (renewable)

**How**: 
- US: USPTO (uspto.gov)
- International: WIPO (wipo.int)

### **2. Copyright Notice**

**Add to All Files**:
```rust
// Copyright (c) 2026 Mondoshawan Protocol
// Licensed under the MIT License
```

**Automatic Protection**: Copyright exists automatically
**Registration**: Optional ($35-65, stronger protection)

### **3. Patents** (If Worth It)

**What Can Be Patented**:
- Novel technical innovations
- Specific implementations
- Unique algorithms (if truly novel)

**Challenges**:
- Blockchain patents controversial
- Expensive ($10k-50k+)
- Time-consuming (2-5 years)
- May not be worth it

**Consider If**:
- Truly novel innovation
- Worth $50k+ to protect
- Can enforce (have resources)
